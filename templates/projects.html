The Campiagn Manager

The Campaign Manager (CM) is a new observing mode that I developed for Skynet. The CM semi-autonomously and indefinitely scales and schedules exposures on dozens of robotic telescopes simultaneously while allowing for modification to scaling parameters in real time. The CM is useful for follow up to various transient phenomena including gravitational-wave events, gamma-ray burst localizations, and young supernovae. The CM is primarily written in Python, with database interactions written in SQL. Additionally, I developed a webpage deticated to monitoring and updating observations scheduled with the CM.

Check out my peer-reviewed systems paper for more details: Skynet' New Observing Mode: The Campaign Manager

Polarimeter

I am currently building a robotic V- and I-band polarimeter. This instrument will also be used to measure the polarizations of GRB afterglows more rapidly than has been achieved to date, within tens of seconds of spacecraft detection, when some afterglows are expected to be significantly polarized.

The Skynet/PROMPT polarimeter consists of two rotating half-wave plate/beam-splitter cube assemblies (V- and I-optimized, respectively) on a linear actuator so either can be moved into the optical path. The control software is written in C#. Additional helper scripts are written in Python.

Radio Mapping Algorithm

The radio mapping algorithm contaminant cleans, maps, and photometers radio astronomy data. The details of the project are far too complex to summarize here (see my co-authored 50-page paper below). The algorithm is currently being integrated into Skynet and will automatically process all data taken from Skynet radio telescopes around the world. The algorithm is written in C++ and is multi-threaded to improve run time on large data sets.

Check out my peer-reviewed co-author paper for more details: Skynet Algorithm for Single-Dish Radio Mapping I.

Gravitational Wave Algorithm and Plotter

I'm currently working on an online, open-source gravitational-wave modeling and plotting tool available to both researchers and students alike. Users are able to modify event parameters to match the simulated the data to the real data in order to extract the inclination, mass, and distance for the GW event. We use the LSC Algorithm Library Suite (LALSuite) written in C to model the events, Python on the back-end, and a combination of Typescript, HTML, and CSS for the front-end.

Try out the tool for yourself: Plotting Tool

Multiplayer FPS Game

After years of wanting to build my very own video game, I finally created one last year using the Unity game engine. There was so much more to game development than I had imagined.
